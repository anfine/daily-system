# 第二阶段：个人网页与远程使用

状态：下一阶段

目标：
- 做一个个人网页，主要给自己看
- 把现有 `heatmap` 用作每日 meta 展示
- 让网页上的写字板不再依赖本地 `localStorage`
- 但写字板正文仍通过 agent 保存到本地，降低隐私泄露风险
- 增加管理员登录，只有管理员可以修改网页上的 meta
- 让项目逐步具备 Docker 化与服务器一拉即运行的部署能力

原则：
- 先做读取链路，再做页面整合
- 先做“公开展示”和“管理员修改”边界，再做完整登录细节
- heatmap 先能稳定接入首页，不急着一开始就重写成复杂前后端架构
- 写字板的远程可用性与隐私隔离要同时设计
- 从这一阶段开始，目录、配置和运行方式都按“后续可 Docker 部署”来组织
- 尽量做到服务器执行 `git pull` 后，可以通过统一启动方式直接运行
- 提前明确需要持久化的数据目录，例如 SQLite、`daily_logs`、导出的 JSON
- 敏感配置通过环境变量管理，不把管理员凭据或部署密钥写死在仓库里
