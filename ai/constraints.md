# 设计约束

## 技术约束

1. 后端保持轻量，不引入复杂框架。
2. 数据库使用 SQLite。
3. 不使用 ORM，直接使用标准库 `sqlite3` 或等价轻量方案。
4. 优先复用现有 Flask `agent.py`，不为了“更现代”而额外迁移框架。

## 产品约束

1. 不引入用户系统。
2. 不做多人协作。
3. 不做复杂权限模型。
4. 不做在线富文本编辑器，继续保持当前简单文本输入模式。

## 工程约束

1. 项目应保持单人开发复杂度。
2. 优先渐进改造，不一次性重写前后端。
3. 迁移期间必须保留 Markdown 和 JSON 兼容输出。
4. 文档里的模型语义必须和当前页面导出格式保持一致。

## 数据约束

1. SQLite 是主数据源。
2. `daily_logs/*.md` 是归档产物，不再作为长期主事实来源。
3. `daily_meta_map.json` 和 `daily_char_map.json` 是展示层导出产物。
4. `notes` 属于每天的 entry，不属于某个单独 meta。
5. `done` 是每天某个 meta 的原始业务事实，不应只在导出 JSON 时临时推断。
